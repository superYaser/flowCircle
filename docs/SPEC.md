# SPEC.md - 快闪群聊 App

## 产品概述

| 属性 | 内容 |
|------|------|
| 产品名称 | 快闪群聊（flowCircle） |
| 版本 | v1.0 |
| 目标平台 | iOS / Android 移动端 |
| 产品定位 | 轻量化、即时性、话题驱动的快闪式群聊工具 |

## 核心目标

1. **零门槛参与**：无需加好友，点击即聊
2. **热度可视化**：通过动态效果直观感受群聊活跃度
3. **瞬时体验**：消息限时保存，营造"在场感"

## 功能架构

```
flowCircle/
├── 首页（广场）
│   ├── 动态群聊展示区（游走卡片）
│   ├── 热度可视化系统（呼吸灯效果）
│   ├── 弹幕发送区（底部输入框）
│   └── 快捷操作（+号菜单：建群/修改群名）
├── 群聊房间
│   ├── 消息展示区（上下滚动，5分钟限时）
│   ├── 消息输入区（12字×2行限制）
│   ├── 摘要展开功能（24-50字）
│   └── 群管理（解散群、删除消息）
└── 系统层
    ├── 关键词过滤系统
    ├── 自动任务推送（5个固定群）
    └── 用户系统（注册登录、一群限制）
```

## 详细功能需求

### 3.1 首页（广场页）

#### 3.1.1 群聊展示区（游走卡片）

- **展示形式**：全屏动态游走卡片，群名称以浮动卡片形式在首页背景上游动
- **视觉规范**：
  - 卡片带1px边框（颜色随热度变化）
  - 群名称后方显示实时人数（如"徒步 12人"）
  - 字体大小：普通群16px，用户自己创建的群18px（大2号）
  - 卡片背景半透明，避免完全遮挡背景

#### 3.1.2 热度可视化系统（呼吸灯）

**活跃度计算逻辑**：
- 统计窗口：最近1分钟的消息量
- 数据刷新：每10秒刷新一次

| 活跃度级别 | 消息量 | 边框颜色 | 呼吸频率 | 特殊效果 |
|-----------|--------|----------|----------|----------|
| 轻度活跃 | 3条/分钟 | 蓝色渐变 #4A90E2 → #7BB7F0 | 3秒/次 | - |
| 高度活跃 | 10条/分钟 | 橙色渐变 #F5A623 → #F8E71C | 1.5秒/次 | - |
| 极度活跃 | 30条/分钟 | 红色渐变 #D0021B → #FF6B6B | 0.8秒/次 | 脉动缩放 scale 1.0→1.05 |

**技术实现**：
- CSS动画/Native动画实现呼吸效果
- 新消息触发瞬间高亮闪烁

#### 3.1.3 弹幕消息系统

- **输入位置**：首页底部固定输入框（高度56px）
- **发送规则**：
  - 消息以弹幕形式从右向左滚入首页
  - 弹幕轨迹：随机分布在首页上半部分
  - 弹幕样式：半透明背景 rgba(0,0,0,0.6)，白色文字，圆角16px，字号14px，内边距8px 16px
  - 可见性：仅当前在首页的用户可见
  - 保存时长：3分钟，超时自动消失
  - 防重叠算法：新弹幕优先选择空旷轨迹

#### 3.1.4 +号快捷操作

- **位置**：输入框右侧，圆形按钮（直径40px）
- **展开菜单**（底部弹出）：
  - 未创建群时：【创建群聊】按钮
  - 已创建群时：【修改群名】按钮

**建群流程**：
1. 点击【创建群聊】→ 判断登录状态
   - 未登录：跳转登录/注册页
   - 已登录：检查是否已创建群
   - 已有群：Toast提示"您已有创建的群，可解散后重新创建"
2. 创建页字段：
   - 群名称（必填，最多8个汉字）
   - 确认创建按钮
3. 创建成功：自动进入该群聊页面
4. 限制：一个用户仅能创建一个群

### 3.2 群聊房间

#### 3.2.1 基础架构

- **入口**：首页点击任意群名称卡片
- **人数限制**：单群上限20人（含群主）
- **切换逻辑**：可同时加入多个群，但群聊页一次只能展示一个群

#### 3.2.2 消息展示规则

- **呈现方式**：上下滚动列表（类似公告板），非问答式
- **排序**：时间正序（新消息在下）
- **自己消息高亮**：背景色区别于他人（他人白色，自己浅蓝色#E3F2FD）
- **记录保留**：仅保留最近5分钟的消息（全局5分钟滚动窗口）

#### 3.2.3 消息内容规范

**字数限制**：
- 单行最多12个汉字（或24个英文字符）
- 默认显示最多2行（24字）
- 超出2行时：显示前24字 + 【…展开】，点击展开显示完整内容（最多50字）

**输入交互**：
- 输入框显示实时字数统计（0/24，超过变红）
- 超过24字时提示"将生成摘要"
- 发送按钮在0字时禁用

#### 3.2.4 内容审核与管理

**关键词过滤**：
- 全局敏感词库（可配置）
- 触发策略：消息包含敏感词时，发送者本地显示发送成功，但实际不广播

**群主权限**：
- 长按消息弹出操作菜单：【删除此消息】
- 删除后：该消息在所有成员端消失
- 仅群主可见删除按钮

#### 3.2.5 群解散机制

- **入口**：群聊页右上角菜单 → 【解散群聊】
- **二次确认**："解散后4小时内无法创建新群，确认解散？"
- **解散后状态**：
  - 群立即消失（所有成员被踢出）
  - 首页不再显示该群
  - 群主进入4小时冷却期

### 3.3 系统群（固定热度群）

#### 3.3.1 固定展示

首页永远显示5个系统群：
1. 徒步（户外运动）
2. 钓鱼（垂钓爱好）
3. 股票（财经投资）
4. 麻将（棋牌娱乐）
5. 辅导小孩（亲子教育）

#### 3.3.2 自动任务机器人

**运行机制**：
- 每个系统群对应独立自动任务（Cron Job）
- 任务频率：每5-10分钟（根据热度动态调整）
- 内容来源：搜索相关关键词的最新资讯
- 消息格式：12字×2行限制
- 发送策略：系统机器人身份，灰色系统消息样式

## 非功能需求

### 性能要求
- 首页游走动画使用GPU加速
- 限制同时显示的卡片数量（超出屏幕暂停动画）
- WebSocket维护长连接，确保消息实时推送

### 存储策略
- Redis存储5分钟消息队列，自动过期
- 弹幕3分钟过期

### 安全
- 关键词过滤在服务端和客户端双重校验
- 防逆向破解

### 异常处理
| 场景 | 处理方式 |
|------|----------|
| 网络中断 | 首页显示"离线模式"，自动重连3次 |
| 消息超限 | 输入框实时拦截，超过50字无法继续输入 |
| 群满20人 | 提示"群已满员，请尝试其他群聊" |
| App杀后台 | 5分钟后重新进入，群聊历史清空 |
